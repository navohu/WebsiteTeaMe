User Needs ATOS

Number
  1. Social
  2. Remote
  3. Work on all platforms
  4. To do list
  5. Reminders
Description
  1. The workplace should make people get better in socialising, interacting with other colleagues and bosses. The need of sending messages is for helping people to communicate with each other and to share the parts of the tasks they have done.
  2. The workplace should be remote such that the cost, travel and office expenses for the company decreases and efficiency must increase.
  3. The application...
  4. One of the features of application which will contain list of basic duties of each member.
  5. This is another feature of the application which will be for sending notifications and displaying small messages with the purpose of reminding.
Rationale
  1. The social need is identified by people becoming less social as remote working is increasing and people are therefore not around other people as much. This is an important skill as almost all projects are based on teamwork where you will need to collaborate. The social skill is very important here in order to create the best possible result.
  2. The need of remote working is identified as it requires travel expenses, cost in having offices, electricity and heating.
  3. The need for application work on all platforms is identified because it would be better if the application can be installed on all platforms(IOS, Android), so that it can be used by everyone no need to change the communication device or the platform they have.
  4. The need to have a to do list in the application is identified because it is obvious to that in a team each member is given a task or the list of the things to do. To have this kind of list will help team members get a balanced allocation of tasks for each employee, and feel more accountable.
  5. The reason why this need is identified is today’s busy lifestyle and so a large number of forgetful people. This will be very useful in involving those workers in the project, as well as helping those ones to be much more connected with their team.

Use Case 1

Use Case: Add a New Team

ID: UC1
Brief Description: The users can add a new team
Primary Actors: Users
Secondary Actors: Users invited to the team
Preconditions: Users must be logged in to the system
Main Flow:
  1. Use case starts when user selects “+”
  2. The system shows a new page to create a new team
  3. User types a team name
  4. User types the emails of the users he/she wants to add to the team
  5. User clicks “Done”
  6. The system takes the user to teams page
Post Conditions: The system will add a new team to the list of teams
Alternative Flow: If the invitation email is incorrect notify the user

Use Case 2

Use Case: Send a file

ID: UC2
Brief Description: The users can send pictures and files
Primary Actors: Users
Secondary Actors: Receiving user
Preconditions: Users must be logged in to the system
Files must be in a correct format
Main Flow:
  1. Use case starts when user selects a team
  2. The system takes to the selected team page
  3. Employee chooses “News Feed” by sliding the page to the right
  4. The system displays the selected page
  5. Employee clicks file symbol, adds a file and then “Send”
  6. The file will be seen in the News Feed page
Post Conditions: The system will share files with all members of the group
Alternative Flow: None

Use Case 3

Use Case: Tick done tasks

ID: UC3
Brief Description: The users can tick done tasks
Primary Actors: Users
Secondary Actors: None
Preconditions: Employee must be logged in to the system
There must be tasks in the state “not ready”
Main Flow:
  1. Use case starts when user selects a team
  2. The system displays that team page
  3. Employee chooses “To do list” by sliding the current page
  4. The system takes the employee to tasks page
  5. The employee tick the boxes in front of the tasks he/she has completed
  6. The system saves changes and displays updated done tasks list
Post Conditions: None
Alternative Flow: If an employee does not tick a box in front of the tasks, the list will not be updated
